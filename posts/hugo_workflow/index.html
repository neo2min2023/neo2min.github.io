Hugo_workflow

April 16, 2023

Firstly, start two repos named researchNotes and neo2min2023.github.io on github.

Build a website locally

  1. build site and push to github
# pwd is projects
hugo new site researchNotes
cd researchNotes
git init
git add remote https://github.com/neo2min2023/researchNotes.git
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke themes/ananke
echo "theme = 'ananke'" >> config.toml
hugo server
git add .
git commit -m 'init commit'
git push -u origin master

#

cd .. 
git clone https://github.com/neo2min2023/neo2min2023.github.io.git
cd neo2min2023.github.io
git checkout -b main
touch readme
git add .
git commit -m 'add readme in main branch'


cd ../researchNotes
rm public
git submodule add -b main git@gitee.com:neo2min/researchNotes.git public

hugo

cd public
git remote -v
git add .
git commit -m 'first publishing'
git push origin main
  1. add content and change config.toml
hugo new posts/my-first-post.md
# change draft from true to false
  1. add Katex support

deploy site on github

In the Pages setting of the repo neo2min2023.github.io,

  • choose Deploy from a branch,
  • choose the main branch,
  • from github add a new file named CNAME, add domain name neomin.top and commit
