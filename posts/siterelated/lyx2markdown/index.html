Convert lyx file to latex automatically

Combine lyx and entr, we get a shell command to automatically convert lyx to latex and convert it to markdown file to deploy in hugo.

1ls fmm.lyx | entr -s "lyx -e xetex fmm.lyx && pandoc -f latex -t markdown fmm.tex -o fmm.md && echo markdown file updated"

The default pandoc command convert displayed math in latex as a block enclosed by $$, which causes trouble if the equation is too complicated to render for hugo, for example,

1align environment: $$\begin{aligned} x + y &= 3 \\ x - y &= 4 \end{aligned}$$ is an system of linear equation.

We need the solution in previous post to tackle this problem. A naive idea is to use regex tools like sed to replace all contents between each pair of $$ by either shortcode or code block, i.e.,

1align environment: 
2 
$$\begin{aligned} x + y &= 3 \\ x - y &= 4 \end{aligned}$$ 

3is an system of linear equation.
4
5align environment: 
6```katex
7$$\begin{aligned} x + y &= 3 \\ x - y &= 4 \end{aligned}$$ 
8\`\`\`
9is an system of linear equation.

If the latex document is not too complicated, we can use the following sed command to achieve this goal.

The shortcoming of this approach is that it is not general and prone to error. A better solution is to use pandoc filters, either lua filter or python filter.

Convert latex to markdown using lua filter

From previous understanding of pandoc and panflute, we finally arrive at the following two lua filters named katexMath.lua and katexMath.lua.

 1-- katexMath.lua
 2return {
 3  {
 4    Math = function (elem)
 5      if elem.mathtype == 'DisplayMath' then
 6       return {
 7            pandoc.RawInline('markdown', '\n```katex\n'),
 8            elem,
 9            pandoc.RawInline('markdown', '\n```\n'),
10          }
11      end
12    end,
13  }
14}
15
16-- katexMath2.lua
17return {
18  {
19    Math = function (elem)
20      if elem.mathtype == 'DisplayMath' then
21       return {
22            pandoc.RawInline('markdown', '\n{{\< katex >}}\n'),
23            elem,
24            pandoc.RawInline('markdown', '\n{{\< /katex >}}\n'),
25          }
26      end
27    end,
28  }
29}

To use them as lua filter for pandoc, we can

1pandoc -s -f latex -t markdown -o fmm_lua.md fmm.tex --lua-filter katexMath.lua
2pandoc -s -f latex -t markdown -o fmm_lua2.md fmm.tex --lua-filter katexMath2.lua

Convert latex to markdown with lua filter

There are two basic problem when convert a latex file to markdown that utilizes mathjax to render latex code.

  1. enclose latex equations or symbols between a pair of backticks, this can prevent markdown engine from converting latex code before mathjax.
  2. pandoc convert all align environment in latex, either align or align* into aligned, which can not have equation number for mathjax, thus I need to convert all displayed math equation with \label{some label} to a type of math environment that is rendered as numbered equation by mathjax. Specifically, convert a equation with \label{some label} to \begin{equation} euqation content \label{some label} \end{equation}; and convert an align/aligned/align* environment into \begin{align} euqation content \label{some label} \end{align}. This problem is also discussed here and a solution is proposed as well.

lua filter for converting latex to markdown

 1return {
 2  {
 3    Math = function (elem)
 4      if elem.mathtype == 'DisplayMath' then
 5        if (elem.text:match("\\label")) then
 6          if elem.text:match("\\begin{align") then
 7            elem.text = elem.text:gsub("{aligned}","{align}")
 8          else
 9            elem.text = '\\begin{equation}' .. elem.text .. '\\end{equation}'
10          end
11        end
12        return {
13          pandoc.RawInline('markdown', '\n`\n'),
14          elem,
15          pandoc.RawInline('markdown', '\n`\n'),
16        }
17      elseif elem.mathtype == 'InlineMath' then
18        return {
19            pandoc.RawInline('markdown', ' ` '),
20            elem,
21            pandoc.RawInline('markdown', ' ` '),
22          }
23      end
24    end,
25    Link = function (elem)
26      if elem.attributes["reference-type"] == 'eqref' then
27        return {
28            pandoc.RawInline('markdown','\\eqref{' .. elem.attributes["reference"] .. '}'),
29          }
30      elseif elem.attributes["reference-type"]  == 'ref' then
31        return {
32            pandoc.RawInline('markdown','\\ref{' .. elem.attributes["reference"] .. '}'),
33          }
34      end
35    end,
36  }
